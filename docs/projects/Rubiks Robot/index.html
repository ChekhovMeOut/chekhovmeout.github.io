Summary

Background

I am one of the many fans around the world who became fascinated with the classic twisting puzzle. The high branching factor that arises from just a simple set of rules still astonishes.

After solving the puzzle by hand many times, I decided to start building an automated system. This is not a new idea, but it requires integration between software, hardware, theory, and materials. It has been fairly challenging to navigate such an open-ended design space with limited resources. But I find the entertaining and educational result to be worth it!

Structural

I used Onshape to design several iterations of a custom interface component which docks with the cube, then 3D printed it. The frame is a simple cube structure with one cross brace on each face to mount the motors. I struggled with balancing tolerance requirements and cost constraints. The solution ultimately required assembling the cube around the robot after it had been built.

Electrical Hardware

After experimenting with servo motors and multiplexing, I decided having a dedicated stepper motor and circuit for each face was necessary to achieve the desired speeds. Each of the six NEMA-17 motors is driven by a DRV8824 circuit built on a breadboard. A biphasic wiring system with full step length (1.8Â°) was used. Experimental testing confirmed 600 microseconds was the minimum step delay to consistently avoid skipping, giving a theoretical rotation time of 0.12 seconds. Currently, the mechanical and electrical systems are functioning well. However, motor position drifts over the course of many cycles. In future iterations, this problem could be fixed with an optical encoder.

Software

All of the data structure and and logic is written in Python with code borrowed from several Git repositories. The Python instance is hosted on a Raspberry Pi, which exports commands through GPIO connections to the breadboard. My own version of an A* search algorithm is in progress. The final product will demonstrate the reductions in solving time resulting from improvements like algorithm heuristics, search databases, and reaction time.

Wrapping up

I have learned a great deal about system integration (and the patience it requires). I enjoyed approaching a silly hobby with a technical bend, and was able to develop new skills in programming and electrical hardware.
